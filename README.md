# product-research-automation

1.	After each call
  - Duplicate template-interview.md into ux-research/interviews/.
	- Fill:
	  - front-matter (ID, persona, date, product_area)
	  - Context, Raw notes, Key pains, JTBD
    
2. When a requirement emerges
- Duplicate template-requirement.md into product-research/requirements/.
- Fill the problem, evidence, and link related_interviews.

3. After a research “wave”
- Update the relevant synthesis/*.md file with:
	- list of interviews considered
	- themes
	- mapped requirements.
